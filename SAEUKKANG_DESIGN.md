# 새우깡으로 한계돌파 — Game Design Document

> 이 문서는 게임의 현재 사양을 기록한다. 기능이 바뀌면 변경이력에 한 줄을 추가하지 말고 이 문서의 해당 섹션을 직접 수정한다.
>
> **현재 밸런스 버전: v9** (외침 재충전형 + 풀빌드 30초 보스전 + 코스튬 비교 시스템 + 웨이브 경고 톤 시스템)

## 1. 개요

- **타이틀:** 새우깡으로 한계돌파
- **장르:** 2D 탑다운 로그라이크 불릿 헤븐 (Survivors-like)
- **플랫폼:** HTML5 단일 파일 (`index.html`), PC & 모바일 대응 (`?m=1`로 모바일 강제)
- **핵심 판타지:** 근육질 비둘기가 새우깡을 마구 투척하며 50개 챕터를 돌파한다 — 코스튬으로 자신을 꾸미고 가챠로 등급을 모은다

## 2. 핵심 루프

1. 챕터 진입 → 웨이브 1 → 자동 사격으로 적 처치, XP·드롭·코인 획득
2. XP 누적 → 레벨업 시 3장 카드 중 1장 선택
3. 일정 시간 경과 → 웨이브 2 → 웨이브 3 (밀도와 종류가 점차 강해짐)
4. 웨이브 3 종료 → **챕터 보스** 출현. 보스 격파 시 챕터 클리어 + 코인·코스튬 보상
5. 챕터 클리어 → 챕터 선택 화면으로 복귀, 다음 챕터가 hero 카드로 노출
6. 50번째 챕터(`무한 너머`) 보스 격파 → **시네마틱 엔딩** (전 세계 정복)

## 3. 게임 상태 & 씬 흐름

```
title ─▶ chapterSelect ─▶ playing ─▶ levelup ─▶ playing
   │                          │       │
   ├─▶ shop (skill/costume/skin)      └── paused
   │                          ├─▶ chapterCleared ─▶ chapterSelect | victory
   │                          └─▶ gameover ─▶ title | retry
   └─▶ victory (시네마틱) ─▶ title
```

타이틀과 챕터선택·상점에서도 비둘기 애니메이션이 살아있도록 `state.time`이 진행된다.

## 4. 플레이어 캐릭터: 근육질 비둘기

### 비주얼 사양

`drawPigeon()`이 매 프레임 캔버스 네이티브 드로잉으로 그린다. 타이틀 비둘기도 동일한 함수가 캔버스(`#titlePigeon`)에 렌더링하므로 코스튬이 즉시 반영된다.

| --- | --- |
| 몸통 (32×28 타원, y=10) | `#8B95A1`, 배 하이라이트 `#C9D1DB` |
| 머리 (16r 원, y=−22) | `#8B95A1`, 좌상 하이라이트 `#C9D1DB` (alpha .7) |
| 부리 | `#FFA500` 삼각형 + `#FFC87A` 하이라이트 |

### 코스튬 레이어 순서 (중요)

`drawPigeon` 안에서 z-순서가 정확히 잡혀 있다:

1. 그림자 → 다리 → 몸통 → 가슴 깃털
2. **몸통 코스튬** (옷)
3. 양팔 → 머리 → 부리 → 눈 → 눈썹 → 머리털
4. **머리 코스튬** (모자)
5. 어택 오라
6. (drawPigeon 종료 후) **무기 코스튬** (주먹 위)

이 순서 덕분에 옷이 부리·눈을 가리지 않고, 모자는 얼굴 위에 자연스럽게 얹힌다. 캔버스 변환은 `getTransform`으로 캡처해서 코스튬 호출 시 화면 좌표계로 복귀했다가 다시 비둘기 좌표계로 복원하는 방식이다.

### 웨이브 풀

`waveEnemyPool(wave)`에서 웨이브 1=가벼움, 2=중량감, 3=공세 직전. 챕터 번호에 따라 풀에 위 적이 추가된다. ch30+에는 tyrant×2, ch50에는 tyrant×3까지 누적된다.

### 밀도 & 보상 재밸런스 (탕탕특공대 스타일 + v9 풀빌드 30초 보스전)

화면 가득 잡몹이 쏟아지는 타격감 + "풀빌드 보스 30초" 목표를 동시에 노린다.

- **스폰 레이트** (`spawnRateForWave`): 웨이브 1/2/3 = **5.0 / 7.6 / 10.6** 마리/초 + 챕터당 +2.4% (v0의 ~3× 밀도). v5에서 3.6/5.6/8.0이었으나 v6 이후 풀빌드 도달을 가능케 하기 위해 추가 상향.
- **적 HP** ×0.65, **공격력** ×0.75 (`spawnEnemy`의 `ENEMY_HP_MULT/DMG_MULT`)
- **XP**: `e.xp`는 옛 베이스값 유지, **`gem.value × 0.42`** (`GEM_XP_SCALE`)로 스케일. 풀빌드 도달은 가능하되 wave 단계에서 너무 빨리 풀맥스가 안 되도록 튜닝.
- **보스전 잡몹 스폰**: 웨이브 레이트 ×0.7 + 0.7 스폰 확률 (이전 ×0.5×0.5 = 0.25). 30초 보스전 동안 플레이어가 계속 레벨업하며 풀빌드에 근접.
- 결과: 챕터당 mob 처치 ~2,012마리, 누적 XP ~840 (보스전 포함) → 풀빌드 가까운 카드 ~14-16장 picked.

### 보스 — 6티어 시스템 (5챕터 간격)

웨이브 3 종료 시 1마리 출현. **챕터 구간별로 보스 종류가 다르다**. 같은 베이스 HP 공식 `(3500 + c^1.45 × 110) × chm.hp`에 변종 배수가 곱해진다.

**HP 공식 진화:**
- v0~v3: 단일 외침(30%) 가정. 만빌드 0.4~8초 → 결국 v3에서 3500 베이스 + 외침 1회 30s 목표.
- v4: 외침이 재충전 가능해지면서 ×1.7로 살짝 상향 (cat_king 1.75×).
- v5: 코스튬·먹이·상점 누적 효과를 검수 빠뜨려서 다시 3초 컷 발생.
- **v9 (현재)**: 코스튬 + 먹이(glow) + 상점 풀맥스 + 멀티 무기 + 폭탄/황금 아이템 + 크리 다 곱연산 누적 시 effective DPS ~1500. 풀빌드 = 30초 타겟으로 cat_king 17.0× 까지 상향.

**검증 (ch1, cat_king v9 기준):**
- HP = (3500 + 110) × 1 (chm.hp) × **17.0** = **61,370 hp**
- 외침 1회 = 12% × 61,370 = **7,364 dmg**, 챕터당 2회 캐스트 = 14,729 dmg
- 잔여 HP = 46,641, 풀빌드 effective DPS ~1,500 → **~31초** ✓
- 강한 빌드 (800 DPS): 58초
- 평균 빌드 (300 DPS): 155초 (챕터 cap 480s 안에서 클리어 가능)
- 풀빌드 + 18장 perfect (3,500 DPS): 13초 — 곱연산 누적 시스템상 한계

| --- | --- | --- | --- | --- | --- |
| **왕관 고양이왕** (cat_king) | 1–29 | **17.0** | 1.20 | 1.00 | 돌진 + 5발 털뭉치 부채꼴 (CD 1.8s) |

보스가 화면 밖에 있으면 화면 가장자리에 빨간 화살표 + `보스 N m` 거리 라벨이 떠서 위치를 알려준다. 보스 출현 후 처음 3초 동안 보스 머리 위에 변종 이름 라벨이 표시된다.

### 보스 등장 임팩트 v9

3초컷 시절의 무덤덤한 등장을 "응급 사이렌"급으로 강화:

- **화면 가장자리 빨간 비네트**: radial gradient 펄싱 (5.2Hz strobe)
- **상하 빨간 위험 띠**: 대각 줄무늬 패턴 (hazard stripe)
- **위에서 슬램다운하는 알림 펄**: 0.45s 동안 H×0.6 위에서 떨어짐, ±2.5° 틸트 진동
- **사이렌 SFX (`sfxBossSiren`)**: 220→880Hz 상승 스윕 ×3 + 천둥 노이즈 + sub-bass pulse train
- **카메라 흔들림**: 50 + 240ms 후 32 + 580ms 후 22 (3단 진동)
- **BGM 즉시 전환**: 보스 전용 팩 `BOSS_BGM_PACK` (108ms sawtooth, 디미니시드 코드 진행) — `bgmResync()`로 즉시 교체
- **토스트 배너**: tone='boss' (빨간 4px 보더, 글로우 +44px, 0.42s 펄스)
- **보스 라벨 표시**: '☠ {보스이름} 등장! ☠' (예: '☠ 왕관 고양이왕 등장! ☠')

### 챕터 시간 캡 + 실제 플레이 시간 (`chapterWaveTimes`, `chapterPlayTimeEstimate`)

각 챕터의 플레이 시간 = **웨이브 합계(고정) + 보스전(빌드 의존)**. 캡은 절대 상한이며 보스가 캡을 넘기면 강제 처치.

v9에서 보스 HP 대폭 상향에 맞춰 **약빌드도 보스 처치 시간을 확보**할 수 있도록 캡을 1.3~1.4× 늘렸다.

| 챕터 범위 | 웨이브 1개 | 웨이브 합계 | 보스 여유 | **실제 (만빌드)** | **실제 (보통빌드)** | **실제 (첫시도)** | 캡 |
| --- | --- | --- | --- | --- | --- | --- | --- |

## 9.5 비둘기의 외침 (필살기, v9 재충전형)

**v9부터 챕터당 1회 → 충전 후 무제한 재사용**. 평균 챕터당 2회 캐스트가 자연스럽게 가능하도록 충전 속도와 데미지%를 함께 튜닝.

### 충전 (`ULT_KILL_GAIN = 0.10` per kill)

- 일반 적: +0.10 / 서머너·리퍼: +0.20 (×2) / 폭군·리바이어던: +0.40 (×4)
- 챕터 진행 동안 누적 차오름:
  - 웨이브 1 (5.0/s × 80s): +40 charge (단독으로는 NOT 발동, "1웨이브 벌써 발동" 사용자 피드백 반영)
  - 웨이브 2 (7.6/s × 80s): +60 charge → **첫 100% lands ~mid-wave-2** (cast #1 가능)
  - 웨이브 3 (10.6/s × 80s): +85 charge → **~85% by 보스 spawn** (cast #2가 보스전 진입 직전 거의 fill)
  - 보스전 (30s @ 0.7×0.7 rate): +15 charge → cast #3 불가
- 챕터당 평균 **2회 캐스트** (웨이브2 mid + 보스 spawn 시점)

### HUD 버튼

화면 우하단(모바일은 중앙 하단)에 둥근 버튼. 충전 게이지가 아래→위로 차오르며, 100%가 되면 노란 펄스 후광 + 토스트 배너 "🐦‍🔥 비둘기의 외침 충전!"으로 클릭 유도. v9부터 .used 회색 비활성 상태 제거 — 항상 충전 가능.

### 사용 효과 (`doUltimate`)

- 비둘기 부리가 활짝 벌어지며 `sfxCoo(true, 2.0)` 외침 + 저음 sawtooth(50/80/120Hz) + cooNote 다층 + 고음 triangle 합성으로 "구구~~~" 사운드 v3
- 양 눈에서 화염 분사. drawPigeon이 `state.ultFlashT > 0`인 동안 눈동자→앞쪽으로 두꺼운 주황 화염 콘 + 노란 코어 + 흰 스파크
- `state.camera.shake = 110` (이후 200ms→80, 480ms→60, 800ms→40 4단 페이딩)
- `state.ultScreenFlash = 0.95` 풀스크린 white-hot 플래시 + 220ms 후 0.55 echo
- 보스에게 최대 HP의 **12%** (이전 30%), 모든 잡몹에게 동일 양만큼 피해 → 잡몹 라인 일소
- 토스트 "🐦‍🔥 비둘기의 외침 — 구구~~~!!!" 2.0초

### 데미지 산정 (`ULT_DAMAGE_PCT = 0.12`)

- 1회 캐스트 = 보스 HP의 12%
- 2회 캐스트 = 24% (= 풀빌드 30초 보스전에서 외침이 깎는 비율)
- 나머지 76%는 자력 DPS로 처리 — 외침이 보스를 "녹이는" 도구가 아니라 "기여하는" 도구가 되도록 설계

챕터 시작 시 리셋: `initChapter`에서 `ultCharge = 0`. 보스 처치 시간 설계는 5절 보스 검증 표 참조.

- C/B: 후광 없음
- A: 단일 솔리드 후광 (반경 ×1.15)
- S: **5초 주기 부드러운 반짝임** — bell-curve envelope (peak at 35% of cycle) × 미세 시머. 베이스 0.10 + 0.55 bell으로 0.10~0.65 강도 구간을 부드럽게 오감. v9 이전의 "상시 4Hz 펄스"가 너무 시끄럽다는 피드백 반영하여 시간차 트윙클 방식으로 변경.

**3. 디테일 누적**
- C: 베이스 형태만 (단색, 미니멀)
- B: + 1~2 작은 액센트 (밴드, 스터드)
- A: + 3~4 장식 (보석, 트림, 패치)
- S: + 애니메이션 효과 (회전 별, 펄스, 불꽃, 스파클)

**예시:**
- **h_crown**: C 단순 띠 → B 5점 왕관 → A + 가운데 빨간 보석 → S + 양옆 보석 + 회전 별 + 후광
- **h_beanie**: C 폼 없는 회색 도ome → B 색상+작은 폼 → A 줄무늬+큰 폼 → S 회전 스파클+큰 사이즈+후광
- **b_cape**: C 짧은 회색 뒤판 → B 색상+클래스프 → A 메달리온+상의 트림 → S 거대 황금 망토+애니메이션 흐름+림 스파클
- **b_dragon**: C 회색 비늘 → B + 금색 척추 → A + 배 엠블럼 → S + 비늘 색상 시프트 + 불꽃 위스프
- **w_phoenix**: C 단일 코어 → B + 3 꽃잎 → A + 5 꽃잎 회전 + 후광 → S + 외곽 링 + 카운터 회전 스파클 + 펄스 후광

PC(≥768×700)에서는 오버레이 컨텐츠가 수직 중앙 정렬된다 (`::before`/`::after` flex 스페이서). 모바일은 top-anchored. 백 버튼은 `position:fixed`로 항상 좌상단에 고정.

### 모달 스태킹 규칙

- `.modal`은 `position:fixed; z-index:1000`으로 뷰포트 전체를 덮는다.
- 백 버튼(`.back-btn`)은 `position:fixed; z-index:5` — 일반 상황에서는 보이지만 모달이 떠있으면 가려져야 한다.
- z-index만으로는 Chrome이 일부 뷰포트에서 `position:fixed` 백 버튼을 모달 백드롭 위에 페인트하는 문제가 있어, **MutationObserver**로 어떤 `.modal`이라도 `.hidden`이 풀리면 `body.modal-open` 클래스를 토글한다. CSS는 이 클래스가 있을 때 `.back-btn { visibility:hidden }`로 강제로 가린다. 클릭 흡수와 시각 일관성 모두 확보.

## 15. 배경 (탑뷰 전용)

이 게임은 위에서 내려다보는 탑뷰 로그라이크다. **측면뷰 요소는 절대 사용 금지** (지평선, 세워진 나무, 건물 정면, 깃발 등). 모든 환경 오브젝트는 위에서 본 모습으로 그린다.

### 렌더링 파이프라인

1. **티어 HUE 그라디언트** (screen) — 분위기 컬러 베이스
2. `ctx.translate(-camX, -camY)` — 월드 좌표로
3. **`drawArenaFloor`** (world) — 타일 색 + 그리드 + 타일별 프롭
4. 젬 → 아이템 → 회오리 → 적 → 적탄 → 플레이어(코스튬 포함) → 플레이어 탄 → 파티클 → 데미지 숫자
5. `ctx.restore()` — 스크린 복귀
6. **`drawSkyline`** (screen 오버레이) — 날씨 파티클 + 비네트 + 중앙 ambient glow
7. 보스 경고 배너 + 보스 방향 화살표
