# 새우깡으로 한계돌파 — Game Design Document

> 이 문서는 게임의 현재 사양을 기록한다. 기능이 바뀌면 변경이력에 한 줄을 추가하지 말고 이 문서의 해당 섹션을 직접 수정한다.

## 1. 개요

- **타이틀:** 새우깡으로 한계돌파
- **장르:** 2D 탑다운 로그라이크 불릿 헤븐 (Survivors-like)
- **플랫폼:** HTML5 단일 파일 (`index.html`), PC & 모바일 대응 (`?m=1`로 모바일 강제)
- **핵심 판타지:** 근육질 비둘기가 새우깡을 마구 투척하며 50개 챕터를 돌파한다 — 코스튬으로 자신을 꾸미고 가챠로 등급을 모은다

## 2. 핵심 루프

1. 챕터 진입 → 웨이브 1 → 자동 사격으로 적 처치, XP·드롭·코인 획득
2. XP 누적 → 레벨업 시 3장 카드 중 1장 선택
3. 일정 시간 경과 → 웨이브 2 → 웨이브 3 (밀도와 종류가 점차 강해짐)
4. 웨이브 3 종료 → **챕터 보스** 출현. 보스 격파 시 챕터 클리어 + 코인·코스튬 보상
5. 챕터 클리어 → 챕터 선택 화면으로 복귀, 다음 챕터가 hero 카드로 노출
6. 50번째 챕터(`무한 너머`) 보스 격파 → **시네마틱 엔딩** (전 세계 정복)

## 3. 게임 상태 & 씬 흐름

```
title ─▶ chapterSelect ─▶ playing ─▶ levelup ─▶ playing
   │                          │       │
   ├─▶ shop (skill/costume/skin)      └── paused
   │                          ├─▶ chapterCleared ─▶ chapterSelect | victory
   │                          └─▶ gameover ─▶ title | retry
   └─▶ victory (시네마틱) ─▶ title
```

타이틀과 챕터선택·상점에서도 비둘기 애니메이션이 살아있도록 `state.time`이 진행된다.

## 4. 플레이어 캐릭터: 근육질 비둘기

### 비주얼 사양

`drawPigeon()`이 매 프레임 캔버스 네이티브 드로잉으로 그린다. 타이틀 비둘기도 동일한 함수가 캔버스(`#titlePigeon`)에 렌더링하므로 코스튬이 즉시 반영된다.

| 파트 | 색상 / 세부 |
| --- | --- |
| 몸통 (32×28 타원, y=10) | `#8B95A1`, 배 하이라이트 `#C9D1DB` |
| 머리 (16r 원, y=−22) | `#8B95A1`, 좌상 하이라이트 `#C9D1DB` (alpha .7) |
| 부리 | `#FFA500` 삼각형 + `#FFC87A` 하이라이트 |
| 눈 | 검정 원 + 흰 point. 공격 직전 pupil `#FF6B6B` 발광 |
| 눈썹 | 화난 사선(두께 3.2), `attackPulse`로 강도 변동 |
| 머리털 3가닥 | `#5C6670` stroke 3.5 |
| 양팔(이두/전완/주먹) | `#6D7680/#A8B4C0/#5C6670` 계열 + 손가락 `#2E3440` |
| 다리 / 발 | `#FFA500` 사각형 + 삼각 발 |
| 그림자 | `rgba(0,0,0,.55)` 타원 |

### 코스튬 레이어 순서 (중요)

`drawPigeon` 안에서 z-순서가 정확히 잡혀 있다:

1. 그림자 → 다리 → 몸통 → 가슴 깃털
2. **몸통 코스튬** (옷)
3. 양팔 → 머리 → 부리 → 눈 → 눈썹 → 머리털
4. **머리 코스튬** (모자)
5. 어택 오라
6. (drawPigeon 종료 후) **무기 코스튬** (주먹 위)

이 순서 덕분에 옷이 부리·눈을 가리지 않고, 모자는 얼굴 위에 자연스럽게 얹힌다. 캔버스 변환은 `getTransform`으로 캡처해서 코스튬 호출 시 화면 좌표계로 복귀했다가 다시 비둘기 좌표계로 복원하는 방식이다.

### 웨이브 풀

`waveEnemyPool(wave)`에서 웨이브 1=가벼움, 2=중량감, 3=공세 직전. 챕터 번호에 따라 풀에 위 적이 추가된다. ch30+에는 tyrant×2, ch50에는 tyrant×3까지 누적된다.

### 보스 — 6티어 시스템 (5챕터 간격)

웨이브 3 종료 시 1마리 출현. **챕터 구간별로 보스 종류가 다르다**. 같은 베이스 HP 공식 `(220 + c^1.45 × 46) × chm.hp`에 변종 배수가 곱해진다.

| 변종 | 챕터 | HP × | DMG × | 크기 × | 특수 패턴 |
| --- | --- | --- | --- | --- | --- |
| **왕관 고양이왕** (cat_king) | 1–29 | 1.00 | 1.00 | 1.00 | 돌진 + 3발 털뭉치 부채꼴 |
| **폭군 군주** (tyrant) | 30–34 | 1.20 | 1.10 | 1.05 | 돌진 + 반경 260 쇼크웨이브 (0.6s 텔레그래프) |
| **심해 군주** (leviathan) | 35–39 | 1.35 | 1.10 | 1.10 | 돌진 + 5발 부채꼴 호밍 촉수 (0.5s 텔레그래프) |
| **환영의 군주** (phantom) | 40–44 | 1.55 | 1.18 | 1.00 | 3.5초마다 텔레포트 + 1.8초마다 3연 다크 웨이브 |
| **잿불 불사조** (phoenix) | 45–49 | 1.80 | 1.25 | 1.10 | 10발 화염 링 AOE + **HP 50% 도달 시 1회 부활(60% HP)** |

보스가 화면 밖에 있으면 화면 가장자리에 빨간 화살표 + `보스 N m` 거리 라벨이 떠서 위치를 알려준다. 보스 출현 후 처음 3초 동안 보스 머리 위에 변종 이름 라벨이 표시된다.

### 챕터 시간 캡 (`chapterWaveTimes`)

각 웨이브의 길이와 보스가 살아있을 수 있는 최대 시간이 챕터별로 다르다. 저난도에서는 빠르게 끝나고, 고난도에서는 빌드 구축 여유가 있도록 설계.

| 챕터 범위 | 웨이브 길이 | [w1End, w2End, w3End, cap] | 챕터 캡 |
| --- | --- | --- | --- |
| 1~10 | 80s | [80, 160, 240, 360] | 6분 |
| 11~25 | 130s | [130, 260, 390, 540] | 9분 |
| 26~40 | 170s | [170, 340, 510, 750] | 12분 30초 |
| 41~50 | 200s | [200, 400, 600, 900] | 15분 |

보스가 캡을 초과해도 클리어가 안 되는 경우, 강제 해제(보스 즉사) 처리.

### 난이도 배율 (`chapterMult`)

5단 piecewise 곡선:
- Ch 1~10: 선형 +0.26 hp / +0.08 dmg
- Ch 10~20: 지수 1.09/1.04
- Ch 20~30: 지수 1.13/1.05
- Ch 30~40: 지수 1.10/1.04
- Ch 40~50: 지수 1.08/1.035

코스튬 보너스(S 등급 풀 장착 시 약 +50% 글로벌)를 가정한 곡선이다.

## 7. HUD

좌상단:
- Lv 뱃지 + XP 바
- HP 바
- `#invBar` 획득 업그레이드 인벤토리 — 카드별 아이콘 + `현재/최대` 레벨 pill, 등급별 테두리 색

우상단:
- 📕 챕터 N · 챕터 이름
- 웨이브 N / 3 (보스 시 `BOSS!`)
- ⏱ **현재 웨이브 남은 시간** (mm:ss 형식)
- ⭐ 점수 / 💀 처치 / 🪙 보유 코인

좌하단: ⏸ 일시정지 / 🔊 음소거
우하단: 모바일 부동 조이스틱

토스트 배너(웨이브 전환·HP 회복 등)는 DOM 레이어로 z-index 9에 떠서 인벤토리·아이템 위에 항상 노출된다.

## 10. 메타 진행: 상점

상점 화면에 3개 탭. 코인 바를 모두 공유한다.

### 🌳 스킬 트리

영구 업그레이드. 3개 가지(방어 / 공격 / 민첩) × 3 노드, 모든 노드에 챕터 클리어 해금 조건이 걸려 있다.

| 가지 | 노드 (위→아래) | 해금 |
| --- | --- | --- |
| 방어 | 단단한 깃털 → 은빛 깃털 자석 → 예비 부활 | Ch1 / Ch3 + prev Lv3 / Ch8 + prev Lv3 |
| 공격 | 근육질 한 방 → 빠른 손놀림 → 추가 새우깡 | Ch1 / Ch5 + prev Lv3 / Ch12 + prev Lv3 |
| 민첩 | 잽싼 발걸음 → 행운의 비둘기 → 경험의 깃털 | Ch2 / Ch6 + prev Lv3 / Ch10 + prev Lv2 |

가격은 레벨에 따라 점증. 구매 가능한 노드는 BUY! 펄스 배지로 강조된다.

### 👗 코스튬 가챠 (NEW 배지)

비둘기 코스튬 시스템. 머리·몸통·무기 슬롯에 등급 S/A/B/C 코스튬 1개씩 장착.

#### 슬롯별 효과 (S = 1.0, A = 0.65, B = 0.40, C = 0.20 배수)
- **머리:** maxHp ×(1+0.25m), 자석 ×(1+0.20m), xpMult ×(1+0.12m)
- **몸통:** maxHp +35m, damageReduction +0.12m, regen +0.5m/초
- **무기:** damage ×(1+0.30m), fireRate ×(1+0.12m), projSpeed ×(1+0.18m); **S만 projCount +1**

#### 가챠 가격
- 1회 뽑기: 1,500 코인 (S 2% / A 8% / B 30% / C 60%)
- 10연 뽑기: 12,000 코인 (A 보장)
- 챕터 클리어 보너스: 10% 확률 드롭, 10판마다 페이티 보장 (S 1% / A 5% / B 19% / C 75%)

## 15. 배경 (탑뷰 전용)

이 게임은 위에서 내려다보는 탑뷰 로그라이크다. **측면뷰 요소는 절대 사용 금지** (지평선, 세워진 나무, 건물 정면, 깃발 등). 모든 환경 오브젝트는 위에서 본 모습으로 그린다.

### 렌더링 파이프라인

1. **티어 HUE 그라디언트** (screen) — 분위기 컬러 베이스
2. `ctx.translate(-camX, -camY)` — 월드 좌표로
3. **`drawArenaFloor`** (world) — 타일 색 + 그리드 + 타일별 프롭
4. 젬 → 아이템 → 회오리 → 적 → 적탄 → 플레이어(코스튬 포함) → 플레이어 탄 → 파티클 → 데미지 숫자
5. `ctx.restore()` — 스크린 복귀
6. **`drawSkyline`** (screen 오버레이) — 날씨 파티클 + 비네트 + 중앙 ambient glow
7. 보스 경고 배너 + 보스 방향 화살표
