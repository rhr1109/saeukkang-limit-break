# 새우깡으로 한계돌파 — Game Design Document

> 최종 업데이트: 2026-04-24
>
> 이 문서는 게임 변경 시마다 반드시 동기화한다. 새 기능/밸런스 조정 후에는 해당 섹션을 수정하고 "변경 이력" 하단에 한 줄을 추가한다.

## 1. 개요

- **타이틀:** 새우깡으로 한계돌파
- **장르:** 2D 탑다운 로그라이크 불릿 헤븐 (탕탕특공대 / Survivors-like)
- **플랫폼:** HTML5 단일 파일 (`saeukkang.html`), PC & 모바일 대응
- **핵심 판타지:** 근육질 비둘기가 새우깡을 마구 투척하며 도시·빌딩·하늘에서 벌어지는 난전을 돌파한다

## 2. 핵심 루프

1. 스테이지 진입 → 60~90초 생존
2. 자동 공격(가장 가까운 적)으로 적 처치 → XP 젬(작은 새우깡) / 드롭 아이템 획득
3. XP 누적 → 레벨업 시 3장의 업그레이드 카드 중 1장 선택
4. 스테이지 타이머 종료 → 보스 출현 → 보스 격파 시 스테이지 클리어 (HP 25% 회복 + 보너스 카드)
5. 8스테이지 완료 → 챕터 클리어 → 다음 챕터 or 기록 보기
6. 챕터 30(`한계돌파`) 클리어 → 최종 엔딩

## 3. 게임 상태 & 씬 흐름

```
title ─▶ chapterSelect ─▶ playing ─▶ levelup ─▶ playing
                               │        │
                               │        └── paused
                               ├─▶ cleared (스테이지) ─▶ levelup ─▶ playing
                               ├─▶ chapterCleared (스테이지 8) ─▶ playing(다음 챕터) | victory
                               ├─▶ gameover ─▶ title | retry
                               └─▶ victory (최종) ─▶ title
```

## 6. 챕터 & 난이도 커브

총 **30개 챕터**. 챕터마다 8 스테이지, 누적 240 스테이지.

### 챕터 이름 테이블 (30) — 지명 기반

난이도 라벨("노멀", "하드" 등)은 폐기. 각 챕터는 **장소 테마**를 이름으로 가진다. 난이도 상승은 챕터 번호와 `chapterMult` 배율로만 표현된다.

| 티어 | 챕터 | 이름 |
| --- | --- | --- |
| 0 Day City | 1~5 | 비둘기 광장 · 도시 공원 · 뒷골목 · 시장통 · 옥상 정원 |
| 1 Neon Night | 6~10 | 네온 거리 · 클럽 뒷문 · 심야 하이웨이 · 버려진 빌딩 · 달 아래 도시 |
| 2 Underground | 11~15 | 동굴 입구 · 수정 동굴 · 용암 동굴 · 지하 던전 · 버섯 숲 |
| 3 Ice | 16~20 | 눈보라 · 얼음 호수 · 설산 등반 · 빙하 절벽 · 오로라 정상 |
| 4 Cosmic | 21~25 | 대기권 돌파 · 우주 정거장 · 소행성대 · 성운의 중심 · 블랙홀 경계 |
| 5 Hellfire | 26~30 | 지옥 관문 · 화산 분화구 · 재의 평원 · 심판의 문 · 한계돌파 |

### 챕터 난이도 배율 (`chapterMult`)

| 지표 | 공식 |
| --- | --- |
| 적 HP | `1.62^(c-1)` |
| 적 공격력 | `1 + 0.25*(c-1)` |
| 적 속도 | `1 + min(0.9, 0.055*(c-1))` |
| 점수 배율 | `1.4^(c-1)` |
| XP 배율 | `1 + 0.20*(c-1)` |

예시:

| 챕터 | HP ×  | DMG × | SPD × |
| --- | --- | --- | --- |
| 1 | 1.00 | 1.00 | 1.00 |
| 5 | 6.86 | 2.00 | 1.22 |
| 10 | 73.6 | 3.25 | 1.50 |
| 20 | 5.4k | 5.75 | 1.90 |
| 30 | 398k | 8.25 | 1.90 |

> 후반 챕터는 플레이어가 드롭 아이템·빌드·관통/회오리/폭발/번개 조합을 극한까지 끌어올려야 돌파 가능하도록 설계.

### 진행 저장

- localStorage key: `saeukkang_progress_v1`
- 저장 필드: `clearedChapter`, `highScore`, `highChapter`, `highStage`
- 잠금 해제 규칙: 챕터 N 클리어 시 N+1 잠금 해제. 기본은 챕터 1만 해제.

## 7. 드롭 아이템

적 처치 시 랜덤 확률로 드롭. Survivors-like 게임처럼 드롭이 체감될 만큼 자주 떨어진다.

### 아이템 5종

| 아이템 | 아이콘 | 효과 | 주 색상 |
| --- | --- | --- | --- |
| 포카칩 | 💖 | HP 30% 즉시 회복 | `#FF6B9D` |
| 자석 깃털 | 🧲 | 화면의 모든 XP 젬을 즉시 흡수 | `#74C0FC` |
| 꿀꽈배기 폭탄 | 💣 | 반경 260px 폭발, 공격력 ×3.5 | `#FFA94D` |

| 코인 | 🪙 | 점수 `+25 + 챕터×10` | `#FFD43B` |

### 드롭 확률 (2단계 롤)

1단계: 적 종류별 `drop` 확률에 실패하면 아무것도 안 떨어짐. 성공하면 2단계로.
2단계: 아이템 `mix` 가중치로 종류 선택.

| ghost    | 28% | 밸런스형 |

| dog      | 40% | |

| summoner | 90% | golden 25 / bomb 20 (거의 확정) |
| **boss** | heal + golden 확정, magnet/bomb 50% 추가 |

### Pity 시스템

- 5킬 동안 드롭이 없으면 다음 롤 확률 2배
- 10킬 이상 없으면 다음 킬은 **확정 드롭**
- 플레이어가 체감상 "드롭이 안 나와 짜증나는 런"이 없도록 보장

### 자석·수집 UX

- 아이템에도 XP 젬과 동일한 자석 pull 적용 (반경 `player.magnet * 1.15`)
- 수집 반경 34px (기존 28 → 확대)
- 드롭 순간 착지 파티클(6개 sparkle + 아이템 색 ring) 연출로 발생 인지 상승

## 11. 배경 (아트 디렉션) — 탑뷰(Top-Down) 전용

**중요:** 이 게임은 위에서 내려다보는 탑뷰 로그라이크다. 따라서 지평선/설산/세워진 나무/건물 정면/깃발 같은 **측면뷰 요소는 절대 사용 금지**. 모든 환경 오브젝트는 "위에서 본 모습"으로 그린다 (원형 분수, 타일, 그림자 있는 벤치 상단, 둥근 나무 캐노피 등).

### 렌더링 파이프라인

1. **티어 HUE 그라디언트** (screen) — 분위기 컬러 베이스 (floor가 대부분 덮음)
2. `ctx.translate(-camX, -camY)` — 월드 좌표로 전환
3. **`drawArenaFloor`** (world) — 타일 색으로 완전히 깔고, 그리드, 타일별 프롭
4. 젬, 아이템, 회오리, 적, 적탄, 플레이어, 플레이어 탄, 파티클, 데미지 숫자
5. `ctx.restore()` — 스크린 좌표로 복귀
6. **`drawSkyline`** (screen 오버레이) — 날씨 파티클 + 비네트 + 중앙 ambient glow
7. 보스 경고 배너
