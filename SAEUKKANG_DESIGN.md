# 새우깡으로 한계돌파 — Game Design Document

> 이 문서는 게임의 현재 사양을 기록한다. 기능이 바뀌면 변경이력에 한 줄을 추가하지 말고 이 문서의 해당 섹션을 직접 수정한다.
>
> **현재 빌드 버전: v12.18** (사용자 v12.17 후속 명령 8건 일괄 반영:
> 1) **난이도 단조증가 보정** — 이전 Ch50 void(16.0/1.70) → Ch51 pigeon_warlord(13.5/1.35) 보스 HP 13% 회귀 버그. pigeon_warlord 17.0/1.70, oracle 17.5/1.75, general 18.0/1.80, emperor 18.5/1.90 으로 단조증가. star_devourer~omega dmgMult 1.55~1.85 → 1.95~2.25, hell_houndmaster~hellgate_sovereign dmgMult 1.70~2.00 → 2.30~2.65 까지 일괄 상향해 expedition 경계 회귀 제거.

## 1. 개요

- **타이틀:** 새우깡으로 한계돌파
- **장르:** 2D 탑다운 로그라이크 불릿 헤븐 (Survivors-like)
- **플랫폼:** HTML5 단일 파일 (`index.html`), PC & 모바일 대응 (`?m=1`로 모바일 강제)
- **핵심 판타지:** 근육질 비둘기가 새우깡을 마구 투척하며 **3개의 원정·총 100개 챕터**를 돌파한다 — 코스튬으로 꾸미고 가족(아내·아기 비둘기)을 만나 함께 우주를 정복한다.

## 2. 핵심 루프

1. 챕터 진입 → 웨이브 1 → 자동 사격으로 적 처치, XP·드롭·코인 획득
2. XP 누적 → 레벨업 시 3장 카드 중 1장 선택
3. 일정 시간 경과 → 웨이브 2 → 웨이브 3 (밀도와 종류가 점차 강해짐)
4. 웨이브 3 종료 → **챕터 보스** 출현. 보스 격파 시 챕터 클리어 + 코인·코스튬 보상
5. 챕터 클리어 → 챕터 선택 화면으로 복귀, 다음 챕터가 hero 카드로 노출
6. 원정 끝(ch50/ch70/ch100) 보스 격파 시 → **원정별 고유 시네마틱**. ch100은 게임의 최종 엔딩.

## 2-1. 원정 시스템 (v12)

100개 챕터를 3단계로 나누어 각 단계의 끝에 고유 클라이맥스를 둔다. 진척도에 따라 다음 원정 탭이 잠금 해제된다.

| 원정 | 이름 | 챕터 | 테마 | 동료 합류 | 잠금 해제 조건 |
| --- | --- | --- | --- | --- | --- |
| Ⅰ | 🌍 지구 정복 | 1–50 | 도시·네온·동굴·얼음·우주·지옥·심연·수정·재·차원 너머 | — | 기본 보유 |
| Ⅱ | 🪐 비둘기 행성 정복 | 51–70 | Pigeon Planet — 깃털 평원·둥지 마을·황금 둥지·행성 코어·황제의 옥좌 | **아내 비둘기** (ch51 클리어 시) · **아기 비둘기** (ch60 클리어 시) | ch50 클리어 |
| Ⅲ | 🌌 우주 정복 | 71–100 | Cosmos — 행성·항성·웜홀·다중우주·기원의 문·우주의 주재자 | (이미 합류한 가족이 끝까지 동행) | ch70 클리어 |

`EXPEDITIONS` 배열과 `expeditionFor(c)` / `isExpeditionEnd(c)`가 챕터 → 원정 매핑·끝 챕터 판정을 담당한다. 챕터 선택 화면 상단에 `🌍 원정 Ⅰ / 🪐 원정 Ⅱ / 🌌 원정 Ⅲ` 탭이 있고, 잠긴 탭은 opacity 0.45 + 자물쇠 토스트로 안내된다. 챕터 선택을 열면 `defaultExpeditionTab()`이 플레이어의 현재 프론티어 원정으로 자동 이동한다.

## 4. 플레이어 캐릭터: 근육질 비둘기

### 비주얼 사양

`drawPigeon()`이 매 프레임 캔버스 네이티브 드로잉으로 그린다. 타이틀 비둘기도 동일한 함수가 캔버스(`#titlePigeon`)에 렌더링하므로 코스튬이 즉시 반영된다.

| 파트 | 색상 / 세부 |
| --- | --- |
| 몸통 (32×28 타원, y=10) | `#8B95A1`, 배 하이라이트 `#C9D1DB` |
| 머리 (16r 원, y=−22) | `#8B95A1`, 좌상 하이라이트 `#C9D1DB` (alpha .7) |
| 부리 | `#FFA500` 삼각형 + `#FFC87A` 하이라이트 |

### 코스튬 레이어 순서 (중요)

`drawPigeon` 안에서 z-순서가 정확히 잡혀 있다:

1. 그림자 → 다리 → 몸통 → 가슴 깃털
2. **몸통 코스튬** (옷)
3. 양팔 → 머리 → 부리 → 눈 → 눈썹 → 머리털
4. **머리 코스튬** (모자)
5. 어택 오라
6. (drawPigeon 종료 후) **무기 코스튬** (주먹 위)

이 순서 덕분에 옷이 부리·눈을 가리지 않고, 모자는 얼굴 위에 자연스럽게 얹힌다. 캔버스 변환은 `getTransform`으로 캡처해서 코스튬 호출 시 화면 좌표계로 복귀했다가 다시 비둘기 좌표계로 복원하는 방식이다.

### 웨이브 풀

`waveEnemyPool(wave)`에서 웨이브 1=가벼움, 2=중량감, 3=공세 직전. 챕터 번호에 따라 풀에 위 적이 추가된다. ch30+에는 tyrant×2, ch50에는 tyrant×3까지 누적되며, ch51+ 원정 Ⅱ는 베이스 풀(쥐·고양이·개·탱크)에 pigeon_warrior/pigeon_brute가 점진적으로 누적된다 (ch70 풀에는 tyrant도 한 마리 섞여 들어온다). ch71+ 원정 Ⅲ는 베이스 풀 위에 void_drone과 star_eye만 추가되며 — 베이스 풀이 여전히 cat/dog/tank 라인업인 점은 의도된 단순화 (티어별 신규 적 페이스 컷오프 우선).

### 밀도 & 보상 재밸런스 (탕탕특공대 v10 → v11.2 환원)

화면 가득 잡몹이 쏟아지는 타격감 + "풀빌드 보스 30초" 목표를 동시에 노린다.

- **스폰 레이트** (`spawnRateForWave`): 웨이브 1/2/3 = **6.0 / 11.0 / 14.5** 마리/초 + 챕터당 +2.4%. 웨이브 2/3는 진짜 격류가 되어 죽여도 죽여도 새 적이 화면에 들어와 있다.
- **적 HP** ×0.65, **공격력** ×0.75 (`spawnEnemy`의 `ENEMY_HP_MULT/DMG_MULT`)
- **mobHpExtra(c) v11.2**: 잡몹 전용 추가 HP 배수 (보스 미적용). 챕터별 **ch1-2 ×1.0 / ch3-15 ×1.0→×1.7 / ch16-30 ×1.7→×2.3 / ch31+ ×2.3 cap**. v11 초기의 ch26+ ×3.6 cap이 너무 무거워 ("L7 by wave 3 ch10" 피드백) v11.2에서 완화. **progressionMult는 잡몹에서 제거** (탱키한 mouse가 킬 레이트 → XP 레이트 → 레벨링을 막음). 보스에는 progressionMult 그대로 적용.
- **적별 XP는 v11.2에서 v10 원래 값으로 환원** (`ENEMY_DEFS.xp`): mouse 1, cat 2, crow 2, dog 4, tank 8, ghost 3, bomber 4, reaper 10, sniper 5, summoner 14, tyrant 22, leviathan 25, boss 40. 신규 적 — pigeon_warrior 7, pigeon_brute 15, void_drone 6, star_eye 13.
- **`gem.value × 0.42`** (`GEM_XP_SCALE`) — v11에서 0.36으로 줄였다가 "레벨업 안 됨" 피드백으로 v11.2에서 환원.
- **보스전 잡몹 스폰**: 웨이브 레이트 ×0.85 + 0.8 스폰 확률. 보스전 30초 동안도 잡몹이 풍성하게 쏟아진다.

### 드롭 웨이브 게이팅 (v10)

"웨이브 1에 모든 아이템이 모인다 → 너무 빨리 강해진다" 피드백 반영. 강 액티브를 후반 웨이브로 미룬다 (`rollItemDrop`):

- **웨이브 1**: drop chance ×**0.45** (드롭 빈도 절반). bomb / golden 가중치 = **0** (절대 안 나옴), magnet ×0.45.
- **웨이브 2**: drop chance ×**0.78**. bomb 가중치 ×0.45, golden ×0.55.
- **웨이브 3 + 보스**: 풀 가중치 (bomb/golden은 사실상 후반 한정 보상으로 작동).

### 보스 — 22변종 시스템 (4원정 통합, v12.17 5챕터 단위 분할)

웨이브 3 종료 시 1마리 출현. **5챕터마다 보스 종류가 바뀐다**. 같은 베이스 HP 공식 `(3500 + c^1.45 × 110) × chm.hp × variant.hpMult × progressionMult()`에 변종 배수가 곱해진다. progressionMult는 보스에만 적용 (잡몹 미적용) — 같은 챕터를 재플레이할 때 상점 누적 + 챕터 갭에 따라 ×1.0~×6 범위로 보스 HP가 가중된다 (frontier 챕터에선 ×1.0).

**v12.17 — sprite/behavior alias 시스템:** 22 변종 전체에 새 sprite 함수를 추가하면 코드가 폭증하므로 **6개 표준 sprite + behavior** (cat_king/tyrant/leviathan/phantom/phoenix/void) 만 유지하고, 각 변종에 다음 4 메타 필드를 부여:
- `baseStyle` — 어느 표준 sprite 로 그릴지 (e.g. `star_devourer.baseStyle = 'leviathan'`)
- `baseBehavior` — 어느 표준 attack pattern 을 쓸지 (대개 baseStyle 과 동일)
- `hueShift` (deg) — `ctx.filter='hue-rotate(...)'` 로 색상 시프트
- `saturate` (배수) — 채도 보정 (ex. black_hole_lord 는 0.4 로 거의 흑백)

### 보스 등장 임팩트 v10

v9에서는 토스트 + 캔버스 슬램다운 알림 + 머리 라벨 + 비네트 = 4겹의 같은 "보스 등장!" 알림이 동시에 떠서 화면이 뭉개졌다. v10은 정보 채널을 분리:

- **토스트 배너** (`tone='boss'`, 2.6초): 변종 이름 + DANGER 부제 — "이번 보스는 누구인가?"를 한 번에 알린다. v10에서 4초 → 2.6초로 단축 — 같은 화면 영역에 들어오는 HP 바와 시간 분리하기 위함.
- **머리 위 라벨** (3초): 보스 위치를 가리키는 부유 라벨.
- **하단 보스 HP 바** (`#bossHpBar`, NEW v10): 변종 이름 + 큰 빨간 HP 트랙 + 절대값/퍼센트 텍스트. 보스 등장 후 **2.5초 지연** 후 페이드인 (토스트가 사라지면서 자연스럽게 인계받음). 보스가 살아있는 동안 항상 노출. HP < 25%일 때 펄스 애니메이션. 스타일은 souls 게임 헤더 형식.
- **화면 가장자리 빨간 비네트** + **상하 빨간 hazard 띠** (2.8초 풀, 1.6초 페이드): 분위기 전용. 텍스트 없음.
- **사이렌 SFX (`sfxBossSiren`)**: 220→880Hz 상승 스윕 ×3 + 천둥 노이즈 + sub-bass pulse train.
- **카메라 흔들림 3단**: 50 / 240ms후 32 / 580ms후 22.
- **BGM 즉시 전환**: 보스 전용 팩 `BOSS_BGM_PACK` (v10 박진감 ↑ — 82ms sawtooth, 디미니시드 진행, 매 비트 kick + 1·3 비트 octave-up 리드 stab + sub-bass pulse + 오프비트 hi-hat 노이즈) → `bgmResync()`로 즉시 교체.

**제거된 것 (v10)**: 캔버스에서 "☠ 챕터 보스 등장! ☠ + ⚠ DANGER" 큰 슬램다운 핀 — 토스트와 100% 중복이어서 삭제. 비네트와 hazard 띠는 그대로 유지.

### 하단 보스 HP 바 (v10 NEW)

## 9.5 비둘기의 외침 (필살기, v9 재충전형)

**v9부터 챕터당 1회 → 충전 후 무제한 재사용**. 평균 챕터당 2회 캐스트가 자연스럽게 가능하도록 충전 속도와 데미지%를 함께 튜닝.

### 충전 (`ULT_KILL_GAIN = 0.072` per kill, v10 밀도 보정)

- 일반 적: +0.072 / 서머너·리퍼: +0.144 (×2) / 폭군·리바이어던: +0.288 (×4)
- 챕터 진행 동안 누적 차오름:
  - 웨이브 1 (5.0/s × 80s): +40 charge (단독으로는 NOT 발동, "1웨이브 벌써 발동" 사용자 피드백 반영)
  - 웨이브 2 (7.6/s × 80s): +60 charge → **첫 100% lands ~mid-wave-2** (cast #1 가능)
  - 웨이브 3 (10.6/s × 80s): +85 charge → **~85% by 보스 spawn** (cast #2가 보스전 진입 직전 거의 fill)
  - 보스전 (30s @ 0.7×0.7 rate): +15 charge → cast #3 불가
- 챕터당 평균 **2회 캐스트** (웨이브2 mid + 보스 spawn 시점)

### HUD 버튼

화면 우하단(모바일은 중앙 하단)에 둥근 버튼. 충전 게이지가 아래→위로 차오르며, 100%가 되면 노란 펄스 후광 + 토스트 배너 "🐦‍🔥 비둘기의 외침 충전!"으로 클릭 유도. v9부터 .used 회색 비활성 상태 제거 — 항상 충전 가능.

### 사용 효과 (`doUltimate`)

- 비둘기 부리가 활짝 벌어지며 `sfxCoo(true, 2.0)` 외침 + 저음 sawtooth(50/80/120Hz) + cooNote 다층 + 고음 triangle 합성으로 "구구~~~" 사운드 v3
- 양 눈에서 화염 분사. drawPigeon이 `state.ultFlashT > 0`인 동안 눈동자→앞쪽으로 두꺼운 주황 화염 콘 + 노란 코어 + 흰 스파크
- `state.camera.shake = 110` (이후 200ms→80, 480ms→60, 800ms→40 4단 페이딩)
- `state.ultScreenFlash = 0.95` 풀스크린 white-hot 플래시 + 220ms 후 0.55 echo
- 보스에게 최대 HP의 **12%** (이전 30%), 모든 잡몹에게 동일 양만큼 피해 → 잡몹 라인 일소
- 토스트 "🐦‍🔥 비둘기의 외침 — 구구~~~!!!" 2.0초

### 데미지 산정 (`ULT_DAMAGE_PCT = 0.12`)

- 1회 캐스트 = 보스 HP의 12%
- 2회 캐스트 = 24% (= 풀빌드 30초 보스전에서 외침이 깎는 비율)
- 나머지 76%는 자력 DPS로 처리 — 외침이 보스를 "녹이는" 도구가 아니라 "기여하는" 도구가 되도록 설계

챕터 시작 시 리셋: `initChapter`에서 `ultCharge = 0`. 보스 처치 시간 설계는 5절 보스 검증 표 참조.

#### 🔧 코스튬 합성 (v12 phase 2 NEW)

가챠 행 바로 아래에 `🔧 코스튬 합성` 버튼이 항상 노출된다. **같은 등급 미장착 코스튬 3장을 소비해 한 단계 위 등급 1장으로 변환**한다. 미장착 슬롯 외에 듀플 처리 — 잉여 자원을 등급 사다리로 끌어올리는 풀-인 메커니즘.

- 변환 규칙: `COSTUME_COMBINE_NEXT = { C:'B', B:'A', A:'S' }`. **S 등급은 상위 합성 불가.**
- `findCombinableGrades()`가 장착 중 uid를 제외한 보유 코스튬을 등급별로 카운트해 ≥3 인 등급 목록을 돌려준다.
- 버튼 클릭 시 가장 높은 합성 가능한 등급부터 자동 선택 (A×3 → S 우선). 토스트 `✨ A×3 → S 합성!` (critical 톤) + sfxLevelUp.
- 부족 시 `합성할 코스튬 부족` 경고 토스트.
- **결과 코스튬은 정의(슬롯·외형)도 무작위.** 동일 슬롯·동일 외형 보장 없음 — 머리 C 3장을 소비해도 무기 B가 나올 수 있다. 풀은 위 원정 게이팅을 따름.
- **소비 우선순위는 oldest-first** (`progress.costumes` 삽입 순서 기준 처음 발견된 3장). 어떤 카드가 소비되는지 사전 미리보기 UI는 없음 — 향후 개선 여지.

#### 등급별 시각 차별화 (3축 누적)

같은 외형도 등급에 따라 시각적으로 명확히 다르다. 세 축으로 누적 차별화:

**1. 사이즈 차이 (`tierScaleFor`)**
- C: ×0.80 (가장 작음, 입문용)
- B: ×0.95
- A: ×1.12
- S: ×1.30 (가장 큼, 최고급)

머리·무기 코스튬은 전체 스케일이 적용되고, 망토 등 몸통 코스튬의 외부 돌출부(뒤판)도 스케일이 적용된다. 몸통 메인은 비둘기 실루엣 클립 때문에 사이즈는 그대로지만 디테일/후광이 변한다.

**2. 후광 (S/A 전용)**
- C/B: 후광 없음
- A: 단일 솔리드 후광 (반경 ×1.15)
- S: **5초 주기 부드러운 반짝임** — bell-curve envelope (peak at 35% of cycle) × 미세 시머. 베이스 0.10 + 0.55 bell으로 0.10~0.65 강도 구간을 부드럽게 오감. v9 이전의 "상시 4Hz 펄스"가 너무 시끄럽다는 피드백 반영하여 시간차 트윙클 방식으로 변경.

**3. 디테일 누적**
- C: 베이스 형태만 (단색, 미니멀)
- B: + 1~2 작은 액센트 (밴드, 스터드)
- A: + 3~4 장식 (보석, 트림, 패치)
- S: + 애니메이션 효과 (회전 별, 펄스, 불꽃, 스파클)

**예시:**
- **h_crown**: C 단순 띠 → B 5점 왕관 → A + 가운데 빨간 보석 → S + 양옆 보석 + 회전 별 + 후광
- **h_beanie**: C 폼 없는 회색 도ome → B 색상+작은 폼 → A 줄무늬+큰 폼 → S 회전 스파클+큰 사이즈+후광
- **b_cape**: C 짧은 회색 뒤판 → B 색상+클래스프 → A 메달리온+상의 트림 → S 거대 황금 망토+애니메이션 흐름+림 스파클
- **b_dragon**: C 회색 비늘 → B + 금색 척추 → A + 배 엠블럼 → S + 비늘 색상 시프트 + 불꽃 위스프
- **w_phoenix**: C 단일 코어 → B + 3 꽃잎 → A + 5 꽃잎 회전 + 후광 → S + 외곽 링 + 카운터 회전 스파클 + 펄스 후광

#### 구출 친구 5종 세트 코스튬 (v12.18 NEW)

**3개 세트 × 5친구 = 15 코스튬.** 한 세트가 통째로 일괄 구매·일괄 장착되며, 5친구가 모두 같은 세트의 코스튬을 입었을 때만 풀세트 보너스가 발동.

| --- | --- | --- | --- | --- | --- | --- |
| 🔥 **잿불 의장단** | 잿불 망토/흉갑/두건/어깨갑/띠 | ×1.01 | ×1.04 | **+9.3% 친구 DPS** | 1,500 🪙 | 7,500 🪙 |
| ✨ **구원의 합창단** | 천상 망토/옷자락/두건/깃옷/띠 | ×1.015 | ×1.04 | **+12.0% 친구 DPS** | 2,000 🪙 | 10,000 🪙 |

**해금 조건**: 5명 모두 합류 (Ch125 다람쥐까지) 되어야 상점에 노출됨.

**구매 흐름** (`buyRescuedSet(setId)`):
1. 보유 안 된 친구 코스튬 수만큼 `costPerPiece × N` 코인 차감.
2. 5친구 각각의 `unlocked` 에 set의 5 코스튬 push.
3. 각 친구의 `equipped.body` 슬롯에 자동 장착.
4. 일괄 구매 후 즉시 풀세트 보너스 발동.

**장착·해제 토글** (`equipRescuedSet` / `unequipRescuedSet`): 이미 보유한 세트는 재구매 없이 한 번 클릭으로 5명 일괄 장착, 다시 클릭으로 5명 일괄 body 슬롯 해제.

**난이도 영향 (v12.18 검수):**
- 친구 5명 합산 DPS = `player.damage × 1.892` (= 합산 fireRate × damageMult).
- 풀파티 effective DPS 에서 친구 비중 ≈ 12-38% (빌드 의존).
- 최대 set boost (Abyssal +15.9% friend DPS) 적용 시 → **총 파티 DPS +1.5~3% 미미**.
- Ch125+ 보스 HP 기준 4% 빠른 처치. 챕터 캡(15분) 안에서 무시 가능 영향. → 챕터 HP 곡선 추가 조정 불필요로 판정.

사용자 피드백 "아내랑 아기 비둘기는 머리장식이랑 옷을 각각 1개씩 착용할 수 있는데 둘다 합쳐서 1개만 착용가능". 이전 `slot.equipped` 가 단일 string ('bow' 등) 이라 wife.bow + wife.apron 둘 다 보유해도 둘 중 하나만 동시 착용 가능했음.

**렌더링**: `getCompanionCostumeDraws(kind)` 가 body→head 순으로 두 draw 함수 배열 반환. wife/child 본체 그린 후 두 슬롯 코스튬을 모두 덧칠.

**상점 UI**: `머리장식 / 옷` 서브탭 (`.cos-slot-tabs[data-companion]`) 으로 분리. 활성 탭의 코스튬만 그리드에 표시 → 사용자가 "어느 슬롯에 무엇을 골랐는지" 헷갈리지 않도록 명확히 분리.

#### 동료 코스튬 미리보기 정렬 (v12.17)

`drawCompanionCostumePreview(cv, kind, costumeId)` — 카드별 72×88 캔버스에 다음 두 가지가 보장:
- **공통 ground line**: `groundY = cssH - 8 = 80px`. 아내 카드는 발 끝(local y=56), 아기 카드는 그림자 하단(local y=21) 을 ground line 에 정확히 맞춤. 옷이 길거나 짧아도 카드 간 발 위치 동일 → 사용자 명령 "다른 코스튬이랑 지면 위치가 동일하게 해주고".
- **DPI 픽셀 깨짐 방지**: `cv.width=cssW*dpr / cv.height=cssH*dpr` + `cv.style.width/height = '72px'/'88px'` 동시에 명시. `setTransform(dpr,0,0,dpr,0,0)` + `imageSmoothingQuality='high'`. `renderCompanionGrid` 에서는 width/height 를 raw 박지 않고 style 만 set → 한 프레임 raw-buffer 흐림 제거.

#### 동료 피격 효과 동기화 (v12.17)

사용자 명령 "남편 비둘기 파티원들은 모두가 남편이 공격 당하면 공격당한 효과가 모두 같이 적용되게". 이전 버전은 `updateCompanions` 가 `p.hitT * 0.85` 를 참조했지만 `damagePlayer` 가 `p.hitT` 를 한 번도 set 하지 않아 동료 hurt-flash 가 사실상 발동하지 않던 잠재 버그.

수정:
1. `damagePlayer(dmg)` 에서 `p.hitT = 0.35` 직접 set + `for(c of state.companions) c.hitT = 0.35` 즉시 전파.
2. update 루프에서 `p.hitT` decay (`p.hitT = max(0, p.hitT - dt)`).
3. `drawCompanion` 의 `ctx.filter = 'brightness(1.6) saturate(1.4) hue-rotate(-15deg)'` 가 모든 동료(아내·아기·구출 친구들)에게 동일 시각 hurt 신호로 적용.
4. `state.player.hitT` 필드를 `state` 초기화 시 0 으로 명시.

- **보스 전용 팩(`BOSS_BGM_PACK`)** v11 응급 사이렌: 68ms sawtooth (v10 82ms → 추가 17% 빨라짐), vol .18. `pack.isBoss=true` 플래그로 `bgmStart()`가 5층 액션 영화 BGM을 동시 출력:
  1. **60Hz sub-bass kick** (every beat) — 85→45Hz 익스포넨셜 sweep, 0.005s attack, 0.22s decay. 두텁고 묵직한 "쿵". 진짜 클럽 킥 느낌.
  2. **kick top-end click** (every beat) — 0.02s 노이즈 burst. 킥의 엣지.
  3. **Octave-up sawtooth lead stab** (1·3 비트) — 보스 리프를 한 옥타브 위에서 찍어줌.
  4. **Hi-hat noise** (2·4 오프비트) — 0.025s 노이즈로 그루브 추진.
  5. **140→520Hz 사이렌 wail** (매 bar 시작, noteIdx 0) — sawtooth + linear gain envelope. 0.55s에 걸쳐 상승, 분명한 "위험 신호". + bar 시작 snare crack 노이즈로 어택 강조.
  
  진행: D 디미니시드 → Eb half-step 상승 → D 트라이톤 → C# 하강 디미니시드 4-bar. `state.bossSpawned && state.bossAlive`일 때 `currentBgmPack`이 자동 swap → `bgmResync()`로 즉시 교체. 보스 잡으면 다음 챕터의 일반 팩으로 복귀.
